# game-openclaw

Scripts and configuration to install, configure, and run an [OpenClaw](https://github.com/openclaw/openclaw) gateway for the game.

The gateway runs an `ai_director` agent that generates jokes from player messages using the Qwen model.

For production mode (integrated system with game-player and game-director), see the [root README](../README.md#production-mode).

## Prerequisites

- Node.js 22+
- pnpm
- A running Matrix homeserver (see `game-infra/`)
- The `ai_director` user registered on the homeserver — see **Getting the Matrix access token** below

## Getting the Matrix access token

The gateway needs a Matrix access token to log in as `@ai_director:matrix.openclaw.local`. Here's how to obtain one:

**Step 1 — Register the `ai_director` user** (if not already registered):

Using the Synapse admin script inside the Docker container:
```sh
docker exec game-player-synapse register_new_matrix_user \
  -c /data/homeserver.yaml \
  -u ai_director \
  -p 'ai_director_pass' \
  --no-admin \
  http://localhost:8008
```

**Step 2 — Log in to get the access token**:
```sh
curl -s -X POST http://localhost:8008/_matrix/client/r0/login \
  -H "Content-Type: application/json" \
  -d '{"type":"m.login.password","user":"ai_director","password":"ai_director_pass"}' \
  | jq .
```

This returns:
```json
{
  "access_token": "syt_xxxxx",
  "user_id": "@ai_director:matrix.openclaw.local",
  "device_id": "ABCDEFGH"
}
```

Copy the `access_token` value — that's what `install.sh` needs.

> The access token is **not self-defined** — it is generated by Synapse when you log in. It is a random string that proves the gateway is authorized to act as `@ai_director`. If you log in again, Synapse generates a new token; the old one may still work or may be invalidated depending on Synapse configuration.

## Setup

```sh
./install.sh
```

This will:
1. Install openclaw globally via pnpm (if not already installed)
2. Install workspace dependencies (if not already installed from root)
3. Create the `state/` directory with agent config
4. Generate `state/openclaw.json` (gateway port 2027, Qwen model, Matrix channel)
5. Prompt for the Matrix access token for `@ai_director:matrix.openclaw.local`

You can also provide both tokens non-interactively:
```sh
QWEN_API_KEY=sk-xxx... 
MATRIX_ACCESS_TOKEN=syt_xxx... 
./install.sh
```

## Standalone Mode

Starts the OpenClaw gateway directly on port 2027.

```sh
./start.sh   # Start the gateway on port 2027
./stop.sh    # Stop the gateway
```

**Testing:** Visit **http://localhost:2027/webchat** to chat with `ai_director` directly in the browser. Whatever you type, the agent replies with a short joke related to your message.

```

├── start.sh                             # Start gateway (standalone, port 2026)

├── stop.sh                              # Stop gateway (standalone)
├── stop-prod.sh                         # Stop proxy + gateway (production)
├── package.json                         # Proxy dependencies (http-proxy)
├── state/                               # Runtime state (gitignored)
│   ├── openclaw.json                    # Gateway config
│   └── agents/ai_director/agent/AGENTS.md
└── README.md
```
